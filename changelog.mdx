## v1.0.0 - Stable Release

<Update label="2024-10-15" description="v1.0.0" tags={["feature", "breaking"]}>
## New Features

- Added real-time collaboration on documentation pages for teams
- Introduced version history with diff viewer for tracking changes
- New search indexing for instant full-text search across all docs

## Improvements

- Enhanced performance with lazy loading for large documentation sites
- Updated UI components to use the latest design system

## Breaking Changes

- Removed legacy `getDocBySlug` API endpoint; migrate to `fetchDoc`
- Configuration format changed from YAML to JSON

## Bug Fixes

- Fixed rendering issues with nested lists in Markdown editor
- Resolved sync errors during offline mode reconnection
</Update>

## Upgrading to the Latest Version

Follow these steps to upgrade Janek Unon smoothly. You back up your data first to avoid disruptions.

<Steps>
  <Step title="Backup Your Data" icon="database">
    Export your documentation site:

    ```bash
    npx janek-unon export --site my-docs --output ./backup
    ```

    Verify the backup integrity before proceeding.
  </Step>

  <Step title="Update Package" icon="package">
    Install the latest version using your preferred package manager.

    <CodeGroup tabs="npm,yarn,pnpm">
    ```bash
    npm install janek-unon@latest
    ```
    ```bash
    yarn add janek-unon@latest
    ```
    ```bash
    pnpm add janek-unon@latest
    ```
    </CodeGroup>
  </Step>

  <Step title="Run Migrations" icon="settings">
    Apply database migrations:

    ```bash
    npx janek-unon migrate
    ```

    Review the changelog for any manual steps, such as updating config files from YAML to JSON.
  </Step>

  <Step title="Restart Service" icon="refresh-cw">
    Restart your Janek Unon instance and test core features like editing and search.
  </Step>
</Steps>

<Callout kind="tip" default-open="true">
  After upgrading from v0.9 or earlier, update your API calls. Replace `getDocBySlug({slug})` with `fetchDoc({path: slug})`.
</Callout>
